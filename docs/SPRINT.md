Fix: removed `width: max-content` from `.featured-carousel__track` (`assets/css/pages/home.css`) — a flex container with `flex-shrink:0` children already overflows and scrolls correctly without it. Verified via direct DOM measurement (`offsetWidth`/`scrollWidth`) on both a simulated 390px width and a genuine ~430px real mobile browser tab — cards now measure the correct 335px with zero body-level horizontal overflow. Desktop layout unaffected (verified via screenshot) since it uses viewport-relative `flex: 0 0 min(78vw, 520px)`, not a percentage.
